## 🔁 What is Version Control?

Version control is a method of tracking and managing changes to project files. It’s especially important when multiple people are working on the same code or when you want to keep a history of changes you can go back to.

A version control system allows you to:

- keep a history of project development;
- see who changed what;
- revert to a previous version if something goes wrong;
- work on different tasks in parallel;
- merge changes into a unified version.

---

## 🧰 Why Git?

**Git** is the most popular version control system. It’s used in companies, academic research, software development, and even for writing theses.

Advantages of Git:

- fully **distributed** system (changes are stored locally);
- **high speed**, even on large projects;
- the ability to work **offline**;
- flexible **branching system**;
- reliability and **control over every project version**.

---

## 💡 Where is Git used?

- 📦 In software development  
- 🧪 In scientific research (for code, LaTeX documents, data)  
- 🧑‍🏫 In academic projects and student teamwork  
- 📝 When writing theses, coursework, and technical documentation  

---

## 🌳 Branching and Merging: How to Collaborate

Git allows you to create **branches** — separate lines of development where changes can be made without affecting the main version. When the work is complete, the branch can be **merged** into the main one.

Example: you and your classmate are working on the same app. They build the interface, and you write the logic. Each of you has a branch. After testing — you merge.

This approach:

- reduces conflicts;
- allows testing of ideas safely;
- improves code quality and stability.

---

## 🧠 Git Tips for Beginners

- Always **write meaningful commit messages** (explain what changed)
- Make **frequent commits** — it’s easier to roll back
- Use a **.gitignore** file to avoid cluttering the repo with unnecessary files
- Don’t be afraid to **break something** — everything can be restored
